# Project Files

This directory contains persistent project context for agents and future maintenance.

## Read Order

1. `state/current-status.md` for current repository truth
2. `state/recent-changes.md` for recent session history
3. `state/next-steps.md` for the immediate recommended action
4. `product/scope.md` for product boundaries
5. `architecture/overview.md` for implementation structure
6. `product/roadmap.md` for planned direction

## Canonical State Files

- `state/current-status.md` records stable current facts.
- `state/recent-changes.md` records recent material changes.
- `state/next-steps.md` records the best immediate next action.

Do not create parallel session handoff files. Keep ongoing continuity in the canonical state files.
